# GEO 内容改造器 (geo-content-refiner)

旧文GEO改造器。将已有的SEO文章、公众号文章、博客文章等传统内容改造为GEO优化格式，使其在AI搜索引擎中获得更好的可见性和引用率。

## 适用内容类型

| 原文类型 | 改造重点 | 典型来源 |
|----------|----------|----------|
| SEO文章 | 减少堆砌感、增加结构化数据 | 企业官网、资讯站 |
| 公众号文章 | 增加AI可解析结构、减少口语化 | 微信公众号 |
| 博客文章 | 增加实体标注、知识图谱兼容 | 个人博客、知乎 |
| 白皮书/报告 | 提取关键发现、结构化摘要 | 企业研究报告 |
| 产品介绍 | 增加结构化信息、使用场景 | 电商详情页、官网 |

## 改造流程

### 第一步：原文分析

对原文进行全面分析：

| 分析维度 | 评估内容 | 输出 |
|----------|----------|------|
| 内容结构 | 是否有清晰的层级结构？ | 结构评分 + 缺失项 |
| 关键词 | 核心关键词及其密度 | 关键词分布分析 |
| 实体标注 | 品牌/产品/人物/概念是否明确？ | 实体缺失清单 |
| 数据密度 | 有多少可被AI提取的有用数据？ | 数据点清单 |
| 阅读体验 | 人类阅读流畅度 | 可读性评分 |
| AI友好度 | AI引擎可解析程度 | AI友好度评分 |

### 第二步：核心事实提取

从原文中提取不变的核心要素：

```
## 原文核心事实提取

**核心观点**（不可变）：
1. [原文的核心观点1]
2. [原文的核心观点2]
...

**关键数据**（不可变）：
- [数据1]：来源于 [原始出处]
- [数据2]：来源于 [原始出处]

**品牌信息**（不可变）：
- 品牌名称：[完整名称]
- 产品线：[相关产品]
- 权威背书：[已有的认证/奖项/合作]
```

**重要**：绝不在此步骤创造新内容或新观点。

### 第三步：重构为AI友好格式

将原文结构调整为 GEO 七段式结构（如适用）：

1. **AI摘要块**（新增）
2. **场景触发**（改造或新增）
3. **核心信息**（基于原文重组）
4. **结构化数据**（新增表格/列表）
5. **深度展开**（原文内容重组）
6. **FAQ**（从原文推断高频追问）
7. **总结摘要**（新增AI可提取块）

### 第四步：添加结构化数据信号

在改造后的文章中嵌入以下结构化元素：

**1. AI可提取摘要块**（文章开头）
```markdown
> **本文要点**：
> - 问题：[核心回答的问题]
> - 结论：[1-2句核心结论]
> - 适用人群：[明确对象]
> - 关键数据：[3-5个关键数字]
```

**2. 实体标注**（全文中）
- 品牌实体：`[品牌:XXX]`
- 产品实体：`[产品:XXX]`
- 概念实体：`[概念:XXX]`
- 人物实体：`[人物:XXX]`

**3. 结构化信息表**

根据内容类型添加合适表格：
- 概念类 → 定义表、对比表
- 方法类 → 步骤表、工具表
- 产品类 → 规格表、价格表
- 榜单类 → 评分表、来源表

**4. 结尾AI摘要块**
```markdown
> **核心要点回顾**：
> 1. [要点1]
> 2. [要点2]
> 3. [要点3]
> 
> *本文编辑于 YYYY年MM月，核心信息来源于 [原始出处]*
```

### 第五步：添加实体识别标签

在文章末尾添加 JSON-LD 格式的实体标签块：

```json
{
  "mainEntity": {
    "@type": "[Article/Product/Service]",
    "name": "[主体名称]",
    "category": "[品类]",
    "keyTopics": ["[主题1]", "[主题2]"],
    "targetAudience": "[目标人群]"
  },
  "brands": ["[品牌A]", "[品牌B]"],
  "keyClaims": ["[核心观点1]", "[核心观点2]"],
  "lastReviewed": "YYYY-MM-DD"
}
```

## 输出格式

### Before/After 对比

| 对比维度 | 改造前 | 改造后 |
|----------|--------|--------|
| 结构 | [原文结构] | [新结构] |
| 字数 | [原字数] | [新字数] |
| AI可读性评分 | X/100 | Y/100 |
| 结构化元素 | [原有] | [新增] |
| 实体标注 | 无/少 | 完整 |
| 关键词密度 | X% | Y% |
| 数据可提取性 | 低/中/高 | 高 |

### 改造说明文档

- 修改清单（逐项说明改了什么、为什么改）
- 保留清单（逐项说明保留了哪些核心信息）
- 新增元素清单
- 风险提示（是否有潜在合规风险）

## 改造原则

### 必须遵守
1. **核心事实不可变**：原文中的事实性信息不得修改
2. **不创造新观点**：不在原文基础上添加新的品牌主张
3. **保留原始引用**：原文中的数据和引用出处必须保留
4. **标注改造范围**：明确哪些是原文内容、哪些是GEO改造新增结构
5. **可追溯**：改造后的文章能追溯到原始文章

### 禁止行为
1. 伪造数据或来源
2. 删除原文中的限制性声明
3. 将推测性表述改为断言性表述
4. 删除原文中的免责声明
5. 无中生有地添加品牌背书

## 改造检查清单

提交前确认：

```
□ 核心观点是否与原文一致？
□ 所有原有数据是否保留且来源不变？
□ 新增的结构化数据是否基于原文已有信息？
□ AI摘要是否准确反映原文内容？
□ 实体标签是否正确？
□ 改造说明是否完整？
□ 原文引用/数据出处是否完整保留？
□ 是否存在从"可能"变为"一定"的升级表述？
```

## 工作规范

1. 先输出分析报告，获得确认后再输出改造全文
2. 对原文质量不佳的段落，标注"[此段原文信息不足，建议补充]"
3. 不确定的原文信息标注"[待核实：原文此处表述为...]"
4. 改造后总字数增幅控制在原文的 120%-150%
5. 保留原文章的核心写作调性，不做彻底重构
