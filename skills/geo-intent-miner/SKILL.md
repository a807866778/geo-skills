# GEO 意图挖掘器 (geo-intent-miner)

意图挖掘与拓词技能。基于种子关键词，模拟用户在AI搜索引擎上的真实提问路径，系统化扩展为结构化问题集群，支持后续内容规划和监测。

## 核心流程

### 第一步：种子关键词输入

接收以下任一格式输入：
- 单个种子关键词（如 "胶原蛋白肽"）
- 多个种子关键词列表
- 品牌核心标签（来自 geo-question-discovery 输出）
- 竞品关键词列表

### 第二步：意图分类扩展

按以下 5 种搜索意图，将每个种子词扩展为自然语言问题集群：

| 意图类型 | 英文标签 | 问题模板 |
|----------|----------|----------|
| 信息型 | informational | "XX是什么""XX的原理""XX有哪些类型" |
| 导航型 | navigational | "XX官网""XX旗舰店""XX客服电话" |
| 交易型 | transactional | "XX多少钱""XX优惠券""XX哪里买便宜" |
| 对比型 | comparison | "XX vs YY""XX和YY哪个好""XX替代品" |
| 本地决策 | local-decision | "XX附近""XX市哪家好""XX线下体验" |

每个意图类型至少扩展 3 个问题变体，考虑不同表达方式：
- 口语化："这玩意儿到底有用没？"
- 专业化："XX的临床实验数据如何？"
- 场景化："产后修复用XX有效吗？"
- 情绪化："XX是不是智商税？"

### 第三步：追问链构建

针对每个核心问题，构建用户可能的追问链条（3-5步）：

```
种子问题："胶原蛋白肽有用吗？"
├─ 追问1："胶原蛋白肽和胶原蛋白有什么区别？"
│   ├─ 追问2："每天吃多少克有效？"
│   │   ├─ 追问3："吃多久能看到效果？"
│   │   │   ├─ 追问4："停用后效果会反弹吗？"
```

追问链规则：
- 每个追问必须是上一步的自然延伸
- 覆盖从"认知→评估→决策→使用→复购"的完整链路
- 识别每个节点的内容缺口

### 第四步：监测提示词库构建

为每个意图分类生成 AI 平台监测提示词：

**提示词格式**：
```
监测目标：[意图类型] - [核心关键词]
监测平台：[豆包/文心/千问/DeepSeek/Kimi]
提示词："[模拟用户真实提问]"
监测频率：每周/月
重点关注：排名变化/新出现内容/竞品露出
```

生成不少于 15 条监测提示词，覆盖所有5大AI平台。

## 输出格式

### 意图集群地图

以树形结构展示所有扩展问题的层级关系：

```
种子词：[XXX]
├── 信息型 (8个问题)
│   ├── 基础认知
│   │   ├── Q1: ...
│   │   └── Q2: ...
│   └── 深度理解
│       ├── Q3: ...
│       └── Q4: ...
├── 对比型 (6个问题)
│   └── ...
├── 交易型 (5个问题)
│   └── ...
├── 导航型 (4个问题)
│   └── ...
└── 本地决策 (7个问题)
    └── ...
```

### 扩展问题清单

全量问题列表（30+），标注：
- 问题文本
- 意图分类
- 所属追问链 ID
- 语言风格（口语/专业/场景/情绪）
- 搜索量预估等级（高/中/低）

### 监测提示词库

按平台分组的监测提示词表，可直接用于定期品牌监测。

## 工作规范

1. 种子词扩展必须有真实搜索行为依据，避免机械组合
2. 追问链必须符合用户认知逻辑，不可跳步
3. 监测提示词必须模拟真实用户提问方式
4. 输出作为后续内容创作的输入源
