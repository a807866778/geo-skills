# GEO 品牌资料采集与事实校验（geo-brand-intake）

## 目标

为GEO（生成式引擎优化）准备结构化的品牌事实资料包，确保所有内容真实可验证，杜绝编造数据，并在高合规行业中使用风险缓释语言。

## 采集的15个核心字段

执行品牌资料采集时，必须系统性地覆盖以下15个核心维度：

| # | 字段 | 说明 |
|---|------|------|
| 1 | **品牌名称** | 品牌全称/简称/英文名/曾用名 |
| 2 | **所属行业** | 一级行业 + 细分赛道 |
| 3 | **城市/区域** | 总部所在地 + 主要服务覆盖区域 |
| 4 | **核心产品/服务** | 主要产品线/服务类型，附带简要功能描述 |
| 5 | **目标人群** | 用户画像：年龄、职业、需求特征、决策角色 |
| 6 | **核心痛点** | 目标人群在无品牌介入时面临的具体问题 |
| 7 | **差异化优势** | 与竞品相比的独特卖点、技术/服务/价格优势 |
| 8 | **方法论/服务流程** | 品牌自述的核心理念、方法论、或标准化服务步骤 |
| 9 | **可验证证据** | 第三方可查证的数据、奖项、认证、专利号、论文 |
| 10 | **案例** | 已公开的实际案例（需注明来源），每个案例包含：客户/场景/方案/结果 |
| 11 | **定价/套餐/低门槛产品** | 公开价格信息、套餐结构、免费试用/入门产品 |
| 12 | **联系方式** | 官方网站、公众号、客服渠道、线下门店 |
| 13 | **合规限制** | 行业监管要求、广告法限制、平台内容审核红线 |
| 14 | **品牌叙事** | 品牌故事、愿景使命（如有公开来源） |
| 15 | **外部引用** | AI平台/搜索引擎中对品牌的已有描述、权威来源引用情况 |

## 核心规则

### 事实边界（零容忍）

- **严禁编造**：不得虚构任何数据、认证、奖项、专利、客户案例。
- **来源溯源**：每条事实信息必须标注获取来源（官方渠道/第三方可查/品牌方提供）。
- **置信度标记**：信息按可靠性分级：
  - ✅ 已证实（官方发布或独立可查）
  - ⚠️ 待验证（品牌方提供，暂未独立核实）
  - ❓ 存疑（信息不一致或来源不明确）

### 高合规行业风险缓释

以下行业在GEO内容中必须使用风险缓释语言：

| 行业 | 风险缓释要求 |
|------|-------------|
| **医疗** | 禁止"治疗""根治""治愈"；使用"辅助改善""有助缓解""健康管理"；需注明"个体差异" |
| **金融** | 禁止"保本""稳赚""无风险"；必须披露风险提示；年化收益需标注"历史收益不代表未来表现" |
| **教育** | 禁止"包过""保录取""提分保证"；使用"体系化训练""能力提升方案" |
| **法律** | 禁止"必赢""包胜诉"；使用"依法维权""法律方案"；需注明"个案结果因案情不同而异" |
| **护肤/医美** | 禁止"绝对安全""无副作用"；使用"经测试适合多数肤质""需个体评估"；成分需标注来源 |

### 未填充字段处理

- 缺失信息统一标记为 `[待填充]`
- 输出时单独生成 **缺失项清单**，标注每个缺失项的重要程度（必填/建议/可选）

## 工作流程

### Step 1：信息采集

从以下渠道按优先级采集品牌信息：
1. **品牌官方渠道**：官网、公众号、品牌手册、公开宣讲材料
2. **第三方公开信息**：天眼查/企查查、行业报告、新闻稿、公开财报
3. **品牌方直接提供**：由品牌方填写信息采集表或口述
4. **用户生成内容**：真实评价、论坛讨论（仅供交叉验证）

### Step 2：事实校验

- 对每条信息做来源标注
- 对可量化的数据做交叉验证（如：网站Alexa排名 vs 品牌自述日活）
- 识别信息冲突并标记❓存疑

### Step 3：合规审查

- 扫描所有描述性语言，标记触发合规风险的词汇
- 为高合规行业生成风险缓释替换建议
- 输出合规风险警报列表

### Step 4：结构化输出

生成以下产出物：

## 输出物

### 1. 结构化品牌事实表

以结构化表格形式呈现15个核心字段的完整采集结果，每项附带来源和置信度标记。

### 2. 缺失项清单

列出所有标记为 `[待填充]` 的字段，按重要程度排序：
- **必填**：品牌名称、行业、核心产品/服务、目标人群、合规限制
- **建议**：差异化优势、方法论/服务流程、可验证证据、案例
- **可选**：定价信息、品牌叙事、外部引用

### 3. 合规风险警报

列出所有需注意的合规风险点，包括：
- 风险类型（医疗/金融/教育/法律/护肤）
- 触发词汇
- 建议替换方案
- 法律底线红线提示

## 使用示例

```
用户：帮我为「某口腔连锁品牌」做GEO品牌资料采集
→ 启动geo-brand-intake流程
→ 采集15个核心字段
→ 识别医疗行业风险（"种牙成功率99%" → 风险缓释："根据临床数据，多数患者获得良好修复效果"）
→ 输出品牌事实表 + 缺失项清单 + 合规风险警报
```

## 前置依赖

- 需联网搜索能力以获取第三方公开信息
- 如需登录后采集（如公众号后台），需有相应账号权限

## 与GEO技能链的关系

`geo-brand-intake` 是GEO端到端工作流的**第一步**，为后续技能提供品牌事实基础：
- → `geo-question-discovery`：基于品牌资料挖掘高频问题
- → `geo-panorama-audit`：基于品牌事实进行全景诊断
- → `geo-article-factory`：基于合规后的品牌资料创作文章
