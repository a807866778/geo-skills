# GEO Publisher - GEO多平台内容发布引擎

## Skill Overview

GEO多平台内容发布引擎提供4级递进式发布架构，从最简单的直链跳转到全自动API发布，覆盖87个内容平台。每级架构对应不同的自动化程度和技术复杂度，用户可根据需求和平台特性灵活选择。

## 4-Tier Publishing Architecture

### L1 - Direct Link (直链跳转)

```
适用性: 所有87平台
自动化程度: ★☆☆☆☆
技术依赖: 无
```

- 为每个目标平台生成含预填参数的发布页面URL
- 用户点击链接直接打开平台编辑器
- 支持预填标题、摘要（部分平台支持URL参数注入）
- 实现方式: 构建 `platform://post?title=xxx&body=xxx` 风格的跳转链接

**示例输出:**
```
微信公众号: https://mp.weixin.qq.com/cgi-bin/appmsg?t=media/appmsg_edit&...
知乎文章: https://zhuanlan.zhihu.com/write
今日头条: https://mp.toutiao.com/profile_v4/graphic/publish
百家号: https://baijiahao.baidu.com/builder/rc/edit?type=news
```

### L2 - Clipboard (剪贴板适配)

```
适用性: 约65个平台（支持富文本编辑器的平台）
自动化程度: ★★☆☆☆
技术依赖: Markdown解析器
```

- 将Markdown源内容自动转换为各平台原生编辑器格式
- 支持的格式转换:
  - 微信公众号 → 微信格式化富文本
  - 知乎 → 知乎编辑器Markdown
  - 今日头条 → 头条富文本HTML
  - 百家号 → 百度编辑器格式
  - 小红书 → 小红书笔记格式（图文分离）
  - 抖音 → 视频文案+标签
- 一键复制到剪贴板，用户在平台编辑器 Ctrl+V 粘贴
- 自动处理平台特殊格式要求（如公众号不支持Markdown代码块，需转为文本）

**格式转换规则:**
```yaml
headers: "#" -> <h2>  # 公众号最大字号限制
code_blocks: "```" -> 高亮引用块  # 公众号不支持代码块
images: "![]()" -> 保留位置标记  # 提示用户手动上传
links: "[text](url)" -> <a href="url">text</a>
tables: "|---|" -> <table>  # 公众号支持简单表格
```

### L3 - Browser Automation (浏览器自动化)

```
适用性: 约40个平台（标准Web编辑器平台）
自动化程度: ★★★☆☆
技术依赖: Playwright
```

使用Playwright实现浏览器自动化发布流程:

1. **打开浏览器** → 启动Chrome/Chromium实例
2. **导航到平台** → 访问平台发布页面，自动处理登录态
3. **填充内容** → 填入标题、正文、封面图、标签
4. **暂停审核** → **【强制】在发布按钮前暂停，等待用户人工审核**
5. **确认发布** → 用户确认后点击发布/保存草稿

```
发布流程:
┌─────────────────────────────────────────────────────┐
│  [启动] → [导航] → [填充标题] → [填充正文]            │
│    ↓                                                │
│  [上传封面] → [设置标签] → [选择分类] → [草稿/发布]  │
│    ↓                                                │
│  ⏸ [暂停 - 用户审核]                                 │
│    ↓                                                │
│  ▶ [用户确认] → [点击发布] → [截图留证] → [下一个]    │
└─────────────────────────────────────────────────────┘
```

### L4 - API Direct (API直发)

```
适用性: 约15个平台（提供开放API的平台）
自动化程度: ★★★★★
技术依赖: 平台API凭证
```

通过官方API直接发布内容，支持草稿模式和定时发布:

| 平台 | API能力 | 认证方式 |
|------|---------|----------|
| 微信公众号 | 草稿创建/发布 | access_token + appID |
| 掘金 | 文章发布 | cookie/session |
| WordPress | REST API | Basic Auth / JWT |
| CSDN | Open API | api_key |
| 知乎 | 受限API | cookie |

**API发布流程:**
```
1. 验证API凭证有效性
2. 将Markdown转换为平台API要求的格式
3. 调用草稿创建接口
4. 返回草稿链接供用户预览
5. 用户确认后调用发布接口
6. 记录发布URL和时间
```

## Scheduled Publishing (定时发布)

```

## Safety Rules (安全规则)

发布引擎内置以下安全机制，**不可跳过**:

1. **发送前审核**: 每次发布前必须暂停等待用户审核内容。显示完整预览（标题+正文+封面+标签），用户明确确认后才执行发布操作。

2. **默认草稿模式**: 所有L3/L4发布默认保存为草稿而非直接发布。用户需额外操作才能转为正式发布。

3. **平台间隔控制**: 多平台连发时，每个平台发布间隔 ≥30秒。此规则不可绕过，防止触发平台反垃圾机制。

4. **验证码暂停**: 检测到CAPTCHA/验证码时立即暂停自动化流程，通知用户手动完成验证，不可尝试自动绕过。

5. **速率限制**: API调用遵守每个平台的速率限制，遇到429响应用指数退避策略。

6. **失败重试**: 发布失败自动重试最多3次，每次间隔递增（30s → 60s → 120s）。3次失败后标记为失败并通知用户。

7. **发布日志**: 每次发布操作记录完整日志（平台、时间、状态、URL、截图），便于后续追溯。

## Workflow

### Step 1: 加载内容
加载待发布内容：标题、正文（Markdown）、封面图、摘要、标签。

### Step 2: 确认发布计划
展示发布计划：目标平台列表、发布时间、发布级别。

### Step 3: 内容适配
根据各平台要求进行内容格式转换（Markdown → 平台格式）。

### Step 4: 逐平台发布
按发布计划依次执行发布操作，遵循安全规则。

### Step 5: 生成发布报告
汇总所有平台的发布状态、URL、截图，生成发布报告。
