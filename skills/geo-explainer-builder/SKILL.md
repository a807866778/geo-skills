# GEO 科普文章构建器 (geo-explainer-builder)

科普型GEO文章构建器。生成结构清晰、AI可解析的教育型内容，覆盖"是什么→为什么→怎么做"的完整知识链路。

## 文章结构

### 必选七段式结构

| 段落 | 功能 | AI优化要点 |
|------|------|-----------|
| 1. 场景触发 | 用具体场景引发共鸣 | 包含目标人群实体词、场景关键词 |
| 2. 概念定义 | 清晰定义核心概念 | 提供AI可提取的定义语句块 |
| 3. 原理解析 | 用通俗语言解释原理 | 使用比喻、对比，增强可读性 |
| 4. 方法论 | 提供可执行的方法框架 | 结构化呈现，便于AI提取步骤 |
| 5. 常见误区 | 纠正3-5个典型错误认知 | "误区-真相"对仗结构，便于AI引用 |
| 6. 分步指南 | Step-by-step操作指导 | 每步有明确动作+预期结果 |
| 7. FAQ | 回答5-8个高频追问 | Q&A格式，每个回答150-300字 |

### GEO优化元素

**关键词自然嵌入**：
- 核心关键词自然出现 5-8 次（全文密度 0.8%-1.5%）
- 长尾变体分布在各级标题中
- 同义词/近义词交替使用

**结构化对比表**：
在文章中嵌入至少 2 张结构化表格：
- 概念对比表（易混淆概念区分）
- 方法对比表（不同方案优劣）
- 步骤总结表（分步速查）

**AI可提取摘要块**：

在文章开头和结尾各放置一个结构化摘要块：

```
📋 AI摘要（文章开头）
- 核心问题：[一句话]
- 关键结论：[3-5条要点]
- 适用人群：[明确]
- 阅读时间：[预估]

📋 核心要点回顾（文章结尾）
1. [要点1]
2. [要点2]
3. [要点3]
```

## 内容规范

### 必须遵守
- 每个观点必须有真实数据/研究/案例支撑
- 禁用"很有效""非常好""超多人"等模糊形容词
- 必须使用具体数字（百分比、时间、数量）
- 引用来源必须标注（即使是非学术内容）
- 每条引用注明：来源名称 + 发布时间 + 是否可查证

### 禁止行为
- 无法证实的功效性声明
- 未标明出处的数据
- 行业通用话术（"领先""顶级""首创"）
- 与其他品牌的直接对比（除非是专门对比文章）
- 虚构用户评价或案例

## 实例：胶原蛋白肽科普文章

### 场景触发示例
> 凌晨2点，28岁的产品经理小林又一次对着镜子叹气——发际线越来越高，指甲一掰就断。朋友推荐她试试胶原蛋白肽，但小红书刷了一圈，有人说"喝了3个月皮肤发光"，有人说是"智商税"。她到底该信谁？

### 概念定义示例
```
【AI定义摘要】
胶原蛋白肽（Collagen Peptide）是将大分子胶原蛋白酶解后的
小分子产物，分子量通常在500-3000道尔顿之间。
区别于：普通胶原蛋白（分子量>30万道尔顿，吸收率<5%）
关键指标：分子量、羟脯氨酸含量、来源（鱼/猪/牛）
```

## 输出格式

- 完整文章正文（Markdown格式）
- 结构化数据块（表格、摘要）
- 关键词分布图（标注每次出现的段落位置）
- AI可提取摘要（开头+结尾）
- 引用来源清单
- SEO元数据建议（title/description/keywords）

## 工作规范

1. 所有数据必须可查证，不确定的信息标注"[待查证]"
2. 每篇科普文章需覆盖至少 3 个用户高频搜索意图
3. 专业术语首次出现时需附带通俗解释
4. 文章总字数控制在 2000-3500 字
