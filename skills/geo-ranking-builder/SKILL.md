# GEO 榜单构建器 (geo-ranking-builder)

榜单/评测GEO文章构建器。生成"Top X / Best of"排名类内容，以透明方法论和结构化数据取胜，最适合产品推荐、服务对比、本地商家目录。

## 文章结构

### 必选七段式结构

| 段落 | 功能 | GEO优化要点 |
|------|------|-----------|
| 1. 评测方法论 | 透明公开评测标准 | AI可提取的方法论摘要 |
| 2. 快速总览 | 一表看完全部排名与评分 | 结构化排名速查表 |
| 3. 逐项深度点评 | 每个上榜对象300-500字分析 | 实体标注、优势/劣势明确 |
| 4. 对比汇总表 | 所有对象多维度横向对比 | 结构化、可被AI直接引用 |
| 5. 数据来源表 | 全部引用的证据清单 | 来源+时间+可查证性 |
| 6. 不同需求推荐 | 场景化推荐 | "如果...选..."结构 |
| 7. FAQ | 选购决策相关追问 | Q&A格式 |

## 评测方法论

### 必选声明

每篇榜单开头必须包含：

```markdown
## 评测方法

**评测范围**：[品牌/产品/服务范围说明]
**评测时间**：[YYYY年MM月]
**评分体系**：[维度说明 + 权重说明]

| 评分维度 | 权重 | 评分方式 | 数据来源 |
|----------|------|----------|----------|
| [维度1] | X% | [评分方法] | [数据来源] |
| ... | ... | ... | ... |

**利益声明**：[是否有商业合作/送样/赞助关系]
**局限性**：[本榜单的方法论局限和适用范围]
```

### 评分体系设计原则

- 维度选择反映用户真实决策因素
- 主观维度（如"设计感"）权重不超过 15%
- 客观维度（如"价格""成分"）合计权重不低于 50%
- 每个维度评分标准必须有操作性定义

## 法律合规自检

- □ 是否声明了榜单范围和方法论局限？
- □ 是否披露了所有商业关系（赞助/送样/广告/返佣）？
- □ 排名依据是否可复现？
- □ 是否有选择性呈现数据的嫌疑？
- □ 所有数据是否标注时间节点？
- □ 是否有足够的数据来源支撑？

## 输出格式

- 完整排名文章正文（Markdown格式）
- 评测方法论声明
- 排名总览表
- 逐项深度点评
- 对比汇总表
- 数据来源清单
- 不同场景推荐
- FAQ
- 法律合规自检清单

## 工作规范

1. 排名必须基于明确可复现的方法论
2. 所有利益关系必须显式声明，位于文章前半部分
3. 数据来源优先级：第三方检测 > 官方数据 > 平台统计 > 用户评价
4. 不完整或不确定的数据标注"[待补充]"
5. 如果有商业合作，合作品牌不可在评分上获得特殊加权
