# GEO Platform Strategy - 内容分发平台策略

## Skill Overview

GEO内容分发平台策略引擎。基于品牌行业（如新能源、消费品、SaaS）、目标区域（国内/海外/特定省份）和内容类型（深度评测、科普解读、产品对比、排行榜），从预构建的87平台数据库中智能匹配最佳分发平台，输出优先级评分、平台适配建议和4周分发节奏排期。

## Core Capabilities

### 2. AI Platform Ecosystem Awareness

不同AI平台具有不同的内容生态偏好，策略匹配时需纳入考量：

- **文心一言（百度生态）**: 偏好百度系内容源（百家号、百度百科、百度经验）。SEO权重高的百度生态内容优先。
- **通义千问（阿里生态）**: 偏向淘宝内容号、知乎、阿里系媒体。电商种草类内容权重较高。
- **豆包（字节生态）**: 优先引用今日头条、抖音、西瓜视频等字节系内容。短视频相关度高。
- **Kimi**: 偏好长文、高质量深度内容，对知乎、少数派、学术来源友好。
- **元宝（腾讯生态）**: 优先微信公众号、视频号内容，社交传播力强的媒体。

### 3. Matching Algorithm

```
Platform Score = Industry_Relevance × 0.35
               + Audience_Overlap × 0.25
               + Content_Format_Fit × 0.20
               + AI_Ecosystem_Weight × 0.15
               + Authority_Level × 0.05
```

其中 `AI_Ecosystem_Weight` 根据目标AI平台的生态偏好动态调整各媒体权重。

### 4. Output Format

每个推荐平台输出：

```yaml
platform_name: 知乎
category: 垂直测评平台
star_rating: 5
ai_ecosystem_score:
  文心一言: 4
  通义千问: 5
  豆包: 3
  Kimi: 5
  元宝: 3
adaptation_advice:
  title_style: 提问式（"XX值得买吗？实测告诉你答案"）
  content_format: 长文带图 + 实测数据
  keywords: 嵌入3-5个长尾问题关键词
  taboo: 避免纯软文，需有真实体验感
priority: 1
reason: 用户画像与目标受众高度重合，知乎内容在多个AI平台引用率高
```

### 5. 4-Week Distribution Rhythm

```
Week 1 - 基础铺设
  Day 1: 微信公众号 + 百家号（核心首发）
  Day 2-3: 知乎 + CSDN/掘金（内容变体）
  Day 4-5: 抖音/B站/小红书（短视频/图文改编）
  Day 6-7: 地方门户/省级媒体号（权威背书）

Week 2 - 扩散放大
  Day 1-2: 搜索系自媒体矩阵（SEO覆盖）
  Day 3-4: 电商种草平台（转化导向）
  Day 5-7: 中央媒体号 + 垂直测评平台（权威加码）

Week 3 - 深度渗透
  Day 1-3: 长尾平台补发 + 问答平台QA
  Day 4-7: 二次传播监控 + 热点结合

Week 4 - 巩固优化
  Day 1-3: 数据复盘 + 高表现内容加推
  Day 4-7: AI引用监测 + 平台权重优化
```

## Workflow

### Step 1: 收集输入参数

- 品牌名称、行业分类
- 目标受众画像
- 内容类型
- 目标区域
- 目标AI平台
- 预算限制（可选）
- 时间要求（可选）

### Step 2: 平台数据库匹配

在8大类87平台中进行多维度匹配，计算每个平台的综合得分。

### Step 3: AI生态权重调整

根据指定的目标AI平台，调整生态权重评分矩阵。

### Step 4: 生成优先级排序

按综合得分降序排列，Top 5-10平台给出详细的平台适配建议。

### Step 5: 输出分发节奏排期表

生成4周分发日历，标注每个平台的发布时间窗口和优先级。

### Step 6: 生成可视化报告

可选：使用图表展示平台评分分布、AI生态覆盖热力图、分发时间线甘特图。

## Notes

- 平台数据库持续维护更新，新增/下线平台及时同步
- AI生态权重每季度复盘调整（参考AI平台更新公告）
- 地域性强的品牌优先匹配地方门户而非全国性平台
- 涉及敏感行业（医疗、金融）需额外合规检查
