# GEO 执行路线图（geo-execution-roadmap）

## 目标

将GEO全景诊断的发现转化为结构化的30/60/90天分阶段执行计划，确保优化行动有清晰的时间线、可衡量的指标和明确的责任分工。

## 前置输入

执行此技能前需具备：
1. `geo-brand-intake` 输出的品牌事实表
2. `geo-panorama-audit` 输出的全景诊断报告（差距分析 + 机会地图）

## 三阶段路线图

### 第一阶段：基础修复（30天）

**目标**：修复关键差距，建立GEO优化基线

| 任务类别 | 具体任务 | 产出 |
|---------|---------|------|
| **信息补全** | 填补品牌事实表中"必填"级别的缺失字段 | 完整的品牌资料包 |
| **准确度修复** | 针对语义诊断中发现的品牌描述错误 → 在各平台输出纠正性内容 | 纠正内容清单 |
| **核心关键词防御** | 对品牌词（品牌名+子品牌+核心产品名）进行搜索引擎/百科内容优化，确保品牌基础信息在所有平台上一致准确 | 品牌词优化完成 |
| **高风险合规审查** | 确保所有已发布内容符合行业合规要求 | 合规审查通过报告 |
| **基础结构搭建** | 建立品牌官网结构化数据、品牌百科词条（如有）、核心平台品牌主页 | 基础信息结构完成 |

**成功指标**：
- 品牌词召回率 ≥ 80%（在5大AI平台上，品牌名被正确提及的比例）
- 品牌语义准确率 ≥ 90%（品牌描述无误的比例）
- 合规审查通过率 100%
- 必填字段填充率 100%

### 第二阶段：内容矩阵扩展（60天）

**目标**：扩展内容矩阵，积累权威信号

| 任务类别 | 具体任务 | 产出 |
|---------|---------|------|
| **品类词内容建设** | 针对诊断中品牌缺失的品类词，创建GEO优化内容（行业科普+解决方案+案例） | 品类词内容矩阵 |
| **痛点词内容建设** | 围绕用户高频痛点问题，创建问答式/方案式内容 | 痛点词内容矩阵 |
| **场景词内容建设** | 制作场景化内容（不同地域/场景的解决方案） | 场景词内容矩阵 |
| **权威信号积累** | 获取行业媒体引用、KOL合作内容、第三方认证/评测、学术/行业会议曝光 | 外部引用数 +30% |
| **竞品差异化内容** | 针对竞品强势维度，制作品牌差异化专题内容 | 差异点内容集 |
| **多平台内容分发** | 将内容矩阵分发至知乎/公众号/小红书/行业论坛/垂直社区等 | 平台覆盖率提升 |

**成功指标**：
- 品类词品牌出现率 ≥ 40%（品类词提及品牌的比例）
- 痛点词品牌出现率 ≥ 30%
- 场景词品牌出现率 ≥ 30%
- 外部权威引用增长 ≥ 30%
- 竞品提及中品牌相对位置提升（进入品类词回答前3位）

### 第三阶段：体系化运营（90天）

**目标**：建立多平台同步机制和持续监控体系

| 任务类别 | 具体任务 | 产出 |
|---------|---------|------|
| **多平台同步** | 建立品牌信息在各内容平台的统一更新机制，确保信息一致性 | 信息同步SOP |
| **内容日历** | 制定持续内容产出计划（周/月频率），建立内容选题库 | 内容运营日历 |
| **监控体系** | 搭建GEO监控仪表板，定期跟踪召回率/语义准确率/竞品动态/内容源变化 | GEO监控仪表板 |
| **数据迭代** | 基于监控数据，每两周进行一轮内容优化迭代 | 优化迭代记录 |
| **危机预警** | 建立AI负面信息预警机制：当品牌在AI平台出现负面关联时自动告警 | 预警规则配置 |
| **团队培训** | 对品牌方内部团队进行GEO意识培训，建立内容合规自审流程 | 培训完成 |

**成功指标**：
- 品牌词召回率 ≥ 95%
- 品类词品牌出现率 ≥ 60%
- 痛点词品牌出现率 ≥ 50%
- 语义准确率 ≥ 95%
- 竞品优势维度覆盖度 ≥ 70%（在竞品强项维度上有品牌自己的差异化内容）
- 监控体系正常运行，预警延迟 < 24小时

| 数据监控频率 | 每周 | 每周×细粒度 | 每日自动化 |

## 工作流程

### Step 1：诊断报告解读

从 `geo-panorama-audit` 输出中提取：
- 四大维度的评分和差距
- 机会地图中的"速效机会"
- 竞品态势中的品牌劣势维度

### Step 2：任务优先级排序

按以下矩阵排序所有待执行任务：
- **Impact（影响）**：该任务对GEO核心指标的提升幅度（1-5）
- **Urgency（紧急度）**：该差距放任不处理的恶化速度（1-5）
- **Effort（工作量）**：完成所需资源/人天（1-5）
- **Dependency（依赖）**：前置任务完成度

优先执行：高Impact × 高Urgency × 低Effort

### Step 3：阶段性任务拆解

将排序后的任务分配到30/60/90天三个窗口：
- P1（30天）：高紧急+低工作量 → 速效修复
- P2（60天）：高影响+中工作量 → 战略建设
- P3（90天）：体系化+自动化 → 持续运营

## 输出物

### 1. 30/60/90天执行路线图

以甘特图或时间轴形式呈现三阶段任务和里程碑。

### 2. 任务清单（Task Breakdown）

每个阶段的任务清单，包含：
- 任务ID、任务名称、任务描述
- 优先级（P0/P1/P2）
- 预计工作量（人天）
- 负责人
- 依赖项
- 完成标准（DoD）
- 关联的成功指标

### 3. 资源需求表

分阶段估算人力、内容量、外部资源需求。

### 4. 成功指标仪表板

每阶段的核心KPI + 测量方法 + 检视频率 + 达标标准。

### 5. 风险管理清单

识别潜在风险 + 影响评估 + 缓解策略 + 应急预案。

## 使用示例

```
用户：基于全景诊断结果，帮我制定GEO执行计划
→ 前提：已完成geo-brand-intake和geo-panorama-audit
→ 解读诊断报告，提取差距和机会
→ 任务优先级排序
→ 分配到30/60/90天三阶段
→ 输出执行路线图 + 任务清单 + 资源配置 + 成功指标
```

## 前置依赖

- 必须在 `geo-brand-intake` 和 `geo-panorama-audit` 之后执行
- 需要品牌方决策层的参与以确认资源投入

## 与GEO技能链的关系

`geo-execution-roadmap` 是GEO端到端工作流的**第三步**：
- ← `geo-brand-intake`：品牌事实基础
- ← `geo-panorama-audit`：全景诊断结果
- → `geo-article-factory`：按路线图计划执行内容创作
- → `geo-platform-strategy`：按路线图确定各阶段内容分发策略
- → `geo-monitoring`：P3阶段的监控体系建设
